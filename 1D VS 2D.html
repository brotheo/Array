What is an Array?

Think of it as a container that holds multiple values under ONE name

🚫 Without Arrays

DECLARE Name1 : STRING DECLARE Name2 : STRING DECLARE Name3 : STRING DECLARE Name4 : STRING DECLARE Name5 : STRING // 5 separate variables! 😩

✅ With Arrays

DECLARE Name : ARRAY[1:5] OF STRING // ONE variable holds ALL 5! 🎉 // Access: Name[1], Name[2]...
💡 Key Idea: An array stores multiple values of the same data type under one identifier. Each value has an index (position number).

📋 1D Array — One Row of Data

Like a single row of lockers, each with a number

Visual: Student Names Array

Ali[1]
Ben[2]
Cat[3]
Dan[4]
Eve[5]

📝 Pseudocode: Declare & Populate

DECLARE Names : ARRAY[1:5] OF STRING Names[1] ← "Ali" Names[2] ← "Ben" Names[3] ← "Cat" Names[4] ← "Dan" Names[5] ← "Eve"
🔑 Remember: 1D = ONE index → Names[i] • Uses ONE loop to go through

🔄 Looping Through a 1D Array

Use a FOR loop to visit every element one by one

Example 1: Output all names

FOR i ← 1 TO 5 OUTPUT Names[i] NEXT i

Output: Ali, Ben, Cat, Dan, Eve

Example 2: Find total marks

DECLARE Marks : ARRAY[1:5] OF INTEGER Total ← 0 FOR i ← 1 TO 5 Total ← Total + Marks[i] NEXT i OUTPUT "Total: ", Total

Example 3: Input & store values from user

FOR i ← 1 TO 5 OUTPUT "Enter mark for student ", i INPUT Marks[i] NEXT i
🧠 Pattern: FOR i ← 1 TO Length → do something with Array[i] → NEXT i

🔍 1D Array: Search & Count

Common IGCSE exam patterns — finding, counting, max/min

Linear Search

INPUT SearchName Found ← FALSE FOR i ← 1 TO 5 IF Names[i] = SearchName THEN OUTPUT "Found at ", i Found ← TRUE ENDIF NEXT i IF Found = FALSE THEN OUTPUT "Not found" ENDIF

Find Maximum

Max ← Marks[1] FOR i ← 2 TO 5 IF Marks[i] > Max THEN Max ← Marks[i] ENDIF NEXT i OUTPUT "Highest: ", Max

Count values above a threshold

Count ← 0 FOR i ← 1 TO 5 IF Marks[i] >= 50 THEN Count ← Count + 1 ENDIF NEXT i OUTPUT Count, " students passed"

📊 2D Array — A Table of Data

Like a spreadsheet with rows AND columns

Visual: 3 Students × 4 Subjects

Math [,1]Science [,2]English [,3]Art [,4]
Student 1 [1,]85927890
Student 2 [2,]76889582
Student 3 [3,]91738497

📝 Pseudocode: Declare a 2D Array

DECLARE Marks : ARRAY[1:3, 1:4] OF INTEGER // [1:3] = 3 rows (students) // [1:4] = 4 columns (subjects) // Access ONE value: OUTPUT Marks[2, 3] // → 95 (Student 2, English)
🔑 Remember: 2D = TWO indices → Array[Row, Column] • Uses NESTED loops

⚡ 1D vs 2D — Side by Side

The key differences you MUST know for the exam

Feature1D Array2D Array
StructureSingle row (list)Table (rows + columns)
IndicesONE — Array[i]TWO — Array[r, c]
DeclarationARRAY[1:N]ARRAY[1:R, 1:C]
Loop typeSingle FOR loopNested FOR loops
Real exampleList of student namesMarks table (students × subjects)
⚠️ Common Exam Trap: If you see Array[Row, Column] → it's 2D. If Array[i] → it's 1D. Two indices = 2D!

🔄 Nested Loops with 2D Arrays

Outer loop = rows, Inner loop = columns

Output ALL values

FOR Row ← 1 TO 3 FOR Col ← 1 TO 4 OUTPUT Marks[Row, Col] NEXT Col NEXT Row

Row Totals

FOR Row ← 1 TO 3 RowTotal ← 0 FOR Col ← 1 TO 4 RowTotal ← RowTotal + Marks[Row, Col] NEXT Col OUTPUT "Student ", Row, " total: ", RowTotal NEXT Row
🧠 Think of it as: "For each student (row), go through all their subjects (columns)"

🔀 Boolean Logic: AND & OR

How AND / OR change what gets selected from arrays

AND — Both must be TRUE

ABA AND B
TRUETRUETRUE ✅
TRUEFALSEFALSE ❌
FALSETRUEFALSE ❌
FALSEFALSEFALSE ❌

OR — At least one must be TRUE

ABA OR B
TRUETRUETRUE ✅
TRUEFALSETRUE ✅
FALSETRUETRUE ✅
FALSEFALSEFALSE ❌
💡 Easy memory trick: AND = stricter (needs ALL). OR = relaxed (needs ANY one).

🔀 Boolean + 1D Arrays

Using AND / OR inside loops to filter array data

Scenario: Two arrays — Marks[] and Attendance[]

Marks[]
85[1]
42[2]
73[3]
55[4]
91[5]
Attendance[]
90[1]
85[2]
70[3]
60[4]
95[5]

AND: Pass = Mark ≥ 50 AND Attend ≥ 80

FOR i ← 1 TO 5 IF Marks[i] >= 50 AND Attendance[i] >= 80 THEN OUTPUT "Student ", i, " PASSED" ENDIF NEXT i // Results: Student 1 ✅, Student 5 ✅ // Student 2: mark too low // Student 3: attendance too low // Student 4: attendance too low

OR: Support = Mark < 50 OR Attend < 80

FOR i ← 1 TO 5 IF Marks[i] < 50 OR Attendance[i] < 80 THEN OUTPUT "Student ", i, " needs support" ENDIF NEXT i // Results: Students 2,3,4 flagged // Student 2: BOTH conditions true // Student 3: attendance true // Student 4: attendance true

🔀 Boolean + 2D Arrays

Combining nested loops with AND/OR conditions

Scenario: Find students who scored ≥ 80 in Math AND Science

// Marks[Student, Subject] — Col 1=Math, Col 2=Science FOR Row ← 1 TO 3 IF Marks[Row, 1] >= 80 AND Marks[Row, 2] >= 80 THEN OUTPUT "Student ", Row, " excels in both!" ENDIF NEXT Row // Student 1: Math=85 Science=92 → PRINTED // Student 2: Math=76 ❌ → NOT printed (AND needs both) // Student 3: Math=91 ✅ Science=73 ❌ → NOT printed

Scenario: Flag any student with ANY mark below 50 (using OR concept)

FOR Row ← 1 TO 3 NeedHelp ← FALSE FOR Col ← 1 TO 4 IF Marks[Row, Col] < 50 THEN NeedHelp ← TRUE ENDIF NEXT Col IF NeedHelp = TRUE THEN OUTPUT "Student ", Row, " needs support" ENDIF NEXT Row
⚠️ Exam Tip: With 2D + Boolean, read carefully: does the question say "all subjects" (= AND logic) or "any subject" (= OR logic)?

📝 Full Exam-Style Example

Step-by-step walkthrough — the grand total pattern

// Calculate row totals and grand total for a 2D array Total ← 0 FOR Row ← 1 TO MaxRow RowTotal ← 0 // Reset each row FOR Column ← 1 TO 10 RowTotal ← RowTotal + Amount[Row, Column] NEXT Column OUTPUT "Total for Row ", Row, " is ", RowTotal Total ← Total + RowTotal NEXT Row OUTPUT "The grand total is ", Total

Step-by-step breakdown:

StepWhat happensWhy
1Total ← 0Grand total starts empty
2FOR Row ← 1 TO MaxRowOuter loop: visit each student
3RowTotal ← 0⚠️ RESET each row! (common mistake if missing)
4FOR Column ← 1 TO 10Inner loop: each column in that row
5RowTotal ← RowTotal + Amount[Row,Col]Add current cell to row's running total
6OUTPUT RowTotalShow that student's total
7Total ← Total + RowTotalAdd row total to grand total
8OUTPUT TotalShow grand total after ALL rows done

📌 Exam Cheat Sheet

Quick reference — memorise these patterns!

📋 1D Array Pattern

DECLARE Arr : ARRAY[1:N] OF DataType FOR i ← 1 TO N // Use Arr[i] NEXT i

📊 2D Array Pattern

DECLARE Arr : ARRAY[1:R,1:C] OF DataType FOR r ← 1 TO R FOR c ← 1 TO C // Use Arr[r, c] NEXT c NEXT r

🔀 Boolean AND

IF condA AND condB THEN // BOTH must be true // Stricter filter // Fewer results ENDIF

🔀 Boolean OR

IF condA OR condB THEN // ANY one true is enough // Relaxed filter // More results ENDIF
